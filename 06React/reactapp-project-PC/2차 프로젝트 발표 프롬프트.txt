발표 Prompt

0. 구현된 페이지 설명과 코드 설명으로 나누어 설명.

1. 홈페이지 
 - 헤더, 햄버거 메뉴, 컨텐츠, 푸터 구성. 햄버거 메뉴는 기능 미구현.
(컨텐츠 부분 Outlet으로 설정)
 - 홈페이지 구성은 이미지 슬라이드형 메인베너, 채팅 바로가기,
  UI 섹션으로 구성하였습니다.
 - 메인베너는 배경이 5초마다 바뀌도록 구현.
 - 채팅 구현은 파일첨부는 구현하지 못했고, 실시간 채팅과 날짜
까지 구현하였음. 방이름과 아이디별로 사용자가 구분.

2. 회원가입
 - 모든 입력란을 입력하도록 구현하였고, 아이디 중복확인과
비밀번호 확인까지 문제 없을 시에만 회원가입이 되도록 구현.
 - 실제로 천 입력 시 중복이라 나오고, 가입 불가. 비밀번호도
틀릴 시 가입 불가. 
 - 이메일 선택시 자동기입되고 수정할 수 없도록 하였고,
휴대폰 번호는 3자리, 4자리에서 자동으로 넘어가도록 구현.
우편번호 찾기는 DaumApi를 사용하여 선택시 자동으로 입력이
채워지도록.

3. 로그인
 - 로그인 하는 부분과 회원가입까지만 구현. 아이디 찾기, 비밀번호
변경은 현재는 장식.
 - 아이디와 비밀번호가 모두 맞아야만 로그인 완료.
 - 미로그인 상태일때 게시판 이용이 제한되는 부분은 PrivateRoute라는
라우트 컴포넌트를 만들어서 제한되는 컴포넌트를 묶었음. 따라서 로그인
되지 않은 상태로 게시판에 들어가면 알림이 뜨고 로그인창으로 이동.
 - 로그인 하면 상단 탑네비가 바뀌고 회원정보 수정창 이동가능. 그리고
창을 닫았다가 들어와도 자동인식하여 로그인상태 지속.

4. 회원정보 수정창
 - 기존 정보 자동 반영. 비밀번호 확인은 다시 입력해야 수정가능하도록 구현.

5. 게시판
 - 자유, QnA, 자료 게시판으로 분리구성. QnA만 댓글 작성가능하도록.
 - 저는 게시판별로 컴포넌트를 만들지 않고 같은 컴포넌트 내에서 분리하여
 3개의 게시판을 구현.
 - 글쓰기에서는 자료게시판만 파일추가 가능하도록.

 - 자유게시판 : 기본적인 CRUD 가능. 수정페이지에서 제목과 내용만 수정가능.
 - 페이지 번호는 자동으로 계산하여 총 페이지 수와 게시글 No를 계산하도록
구현. 날짜는 오늘 날짜만 시분으로 구현.

 - QnA 게시판에서 댓글은 기본적인 CRUD 가능하고 바로 반영되도록.
 댓글 입력 및 수정은 모달창으로 구현.

 - 자료실 게시판에서는 이미지 파일의 경우 미리보기가 가능. 그 외 파일은
다운로드만 가능. 스토리지 이슈로 이미지파일은 새창에서 직접 다운로드.

6. 채팅방 시현
 - 시현 영상. 파일은 못올림. 텍스트만 가능.

<코드>
특징적인 것들만 설명

1. main.jsx
 - 가장 바깥은 채팅창 404 문제로 HashRouter로 라우팅구성했고,
 그 다음은 로그인 관련 Context Provider로 감쌌음.

2. App.jsx
 - 기능별로 라우팅 구성
 - <PrivateRoute> 컴포넌트로 미로그인 사용자에 대한 block 구현.

3. Register.jsx
 - Reducer 함수 사용.

4. module.css 로 css 구성. 컴포넌트별 적용가능.

5. firebase.js 에서 apiKey 관리. firestore, realtime 등은 import해서 사용.
